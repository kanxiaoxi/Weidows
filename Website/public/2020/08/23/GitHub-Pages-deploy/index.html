文件目录: .github/workflows/deploy.yml

解释:

  • name随意起
  • on:[push]意思是在推送时触发action
  • Build and Deploy就是触发的job名称
  • run-on是运行环境(其自带一些基础的环境,Exp:Node.JS,但是更具体的比如Hexo就不会自带了)
  • steps就是步骤了,可以看出下面有三个步骤(步骤内的操作就不细讲了…)
